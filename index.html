Research Overview

The Quantum Threat

Shor's algorithm renders RSA and elliptic-curve cryptography insecure against cryptographically relevant quantum computers. The harvest-now, decrypt-later threat means adversaries are recording encrypted API traffic today to decrypt retroactively once quantum capability matures.

NIST finalized post-quantum cryptographic standards in 2024: ML-KEM (FIPS 203), ML-DSA (FIPS 204), and SLH-DSA (FIPS 205).

Why Now?

HNDL threat accrues continuously from today regardless of when quantum computers arrive.

Layered Approach

Five-layer architecture addressing transport, certificates, tokens, gateways, and applications.

Cryptographic Agility

Algorithm substitution without redesigning system architecture ensures future-proofing.

Migration Framework

Four-phase phased approach minimizes operational risk and maintains backward compatibility.

Reference Architecture

The proposed five-layer architecture separates API security concerns into distinct layers with individual migration timelines and trust boundaries.

1

Transport Layer

TLS 1.3 with hybrid key exchange (ML-KEM + ECDHE). Protects data in transit between clients and servers.

2

Certificate Layer

X.509 chains with post-quantum signatures (ML-DSA, SLH-DSA). Sequential or composite certificate strategies.

3

Token Layer

JWT and OAuth 2.0 with post-quantum signature algorithms. Algorithm-agnostic migration via header identifiers.

4

Gateway Layer

API gateway and reverse proxy. Convergence point for transport, certificate, and token migrations with algorithm negotiation.

5

Application Layer

Business logic consuming validated requests. Generally minimal changes provided lower layers expose stable interface.

Threat Model

Analysis of adversary classes, protected assets, and security goals for quantum-resilient API systems.

Classical Adversary

Passive eavesdropping and active MITM attacks bounded by present-day computational limits.

Harvest-Now, Decrypt-Later

Adversary records encrypted traffic today for retroactive decryption once quantum capability matures.

Forward Secrecy

Hybrid key exchange ensures session security if either classical or post-quantum component remains unbroken.

Authenticity

Post-quantum signatures protect certificate and token integrity against quantum adversaries.

Compatibility

Backward compatibility during transition ensures classical-only clients function without disruption.

Agility

Configuration-driven algorithm selection enables algorithm rollback and future transitions.

Migration Framework

Four-phase phased migration approach prioritizing transport-layer protection with coordinated certificate and token migration.

Phase 0: Inventory

Catalog all cryptographic dependencies across transport, certificate, token, and gateway layers including third-party libraries and vendor-managed infrastructure.

Phase 1: Hybrid Transport

Enable hybrid TLS key exchange at gateway and load-balancer tiers with classical-only fallback for non-upgraded clients.

Phase 2: Certificate & Token Migration

Issue composite or parallel post-quantum certificate chains; begin dual-algorithm JWT issuance and validation.

Phase 3: Agility Consolidation

Remove deprecated classical-only code paths once client population and compliance requirements permit, retaining configuration-level agility.

Publication

Post-Quantum Secure API Communication: Architectures and Migration Strategies for Quantum-Resilient Systems

Author: Subhajit Roy
Year: 2026
Publisher: Zenodo
DOI: 10.5281/zenodo.20413115
Abstract: The advent of cryptographically relevant quantum computers threatens the asymmetric primitives — RSA, Diffie-Hellman, and elliptic-curve cryptography — that underpin the confidentiality and integrity of contemporary API communication. While large-scale quantum adversaries do not yet exist, the "harvest now, decrypt later" threat model means that data encrypted today under classical schemes may already be exposed to future cryptanalysis. This paper examines the practical migration of API security architectures toward post-quantum cryptography (PQC), focusing on the NIST-standardized algorithms ML-KEM (FIPS 203), ML-DSA (FIPS 204), and SLH-DSA (FIPS 205). We propose a layered reference architecture for quantum-resilient API systems, evaluate hybrid key-exchange strategies for TLS 1.3, and analyze the downstream effects of PQC adoption on certificate chains, JWT, OAuth 2.0 token issuance, and API gateway design. We further present a phased migration framework intended to guide practitioners through the transition without requiring a disruptive, single-step cutover.
